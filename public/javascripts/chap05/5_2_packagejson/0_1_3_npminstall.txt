npm audit는 패지지의 알려진 취약점을 검사할수이는 명령어이다.
npm에는 워낙많은 패키지를 관리하기때문에 악성코드를 담고있을수도있다.
이런 악성코드들이 npm에 보고되는데 npm audit 명령을통해
악성코드가 포함된 패키지를 설치하지 않았는지 확인할수있다.

npm audit fix 명령을 입력하면 npm이 스스로 수정할수있는 취약점을 알아서 수정해준다.
주기적으로 npm audit fix 명령을 실행하면 취약점을 알아서 수정해준다.


package.json을 열어보면 dependencies에 설치한 express package, package version정보를 알수있다.
추가적으로 node_modules 폴더도 생겼다. 그안에 설치한 package정보가 들어있다.
분명히 express package하나만 설치했을뿐인데, 여려개의 의존성package들이 들어있다.
그패키지들은 또 다른 package를 의존한다. 이렇게 복잡하게 얽혀있어서 package.json이 필요한것이다.

잘보면 package-lock.json파일도 생성되어있다. 내용을 보면 express외에도 node_module에 들어있는
package들의 정확한 버전과 의존관계가 담겨있다.
npm으로 package를 설치 수정 삭제 할떄마다 package들간의 정확한 의존관계를 이파일에 저장해둔다.
즉 package.json파일은 직접 package를 설치한 기록이고, package-lock.json은 package들간의
의존관계를 명시한 파일이다.
